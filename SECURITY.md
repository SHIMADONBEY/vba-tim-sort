## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 0.x   | :x:                |

## Reporting

- Do NOT disclose sensitive details in a public issue or pull request.
- This project accepts vulnerability reports only via GitHub Private Vulnerability Reporting (repository security advisories). Reports sent by email or other channels will not be processed.
- To report: use the repository's "Security" → "Advisories" → "Report a vulnerability" / "New draft security advisory" flow.

## What to include

- Affected version(s) / tag
- Short summary of the issue and potential impact
- Reproduction steps or PoC (or minimal example)
- Environment (VBA / Excel version, Using OS, platform)
- Attachments, logs, screenshots, and PoC if available


## Response and timeline

- We will acknowledge receipt within 4 days.
- We will coordinate privately via the advisory to develop and verify a fix and decide disclosure timing.
- When a patch is ready we aim to publish a security advisory and a public notice (or provide mitigation steps) together with the fix.

## Disclosure and credits

- We prefer coordinated disclosure. Maintainers will publish an advisory when a fix is available.
- Reporters may be offered credit in the advisory; we will obtain consent before publishing identifying information.

## Privacy

- We will not share reporter identity or contact details without consent, except where required by law.

## Thank you

Thank you for responsibly reporting issues.
