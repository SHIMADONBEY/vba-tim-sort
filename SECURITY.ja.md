| ------------ | ------------------ |
| 1.0.x        | :white_check_mark: |
| < 0.x        | :x:                |

## 報告について

- 機微な情報を公開Issue や Pull-Request として記載しないでください。
- 本プロジェクトでは、脆弱性報告は GitHub の Private Vulnerability Reporting（リポジトリのセキュリティアドバイザリ）からのみ受け付けます。メールやその他の手段による報告は処理しません。
- 報告手順: リポジトリの「Security」→「Advisories」→「New draft security advisory」から報告してください。

## 報告に含めてほしい情報

- 影響を受けるバージョン / タグ
- 問題の概要と想定される影響
- 再現手順またはPoC (可能であれば最小例)
- 実行環境 (Excel のバージョン、利用OS等)
- スクリーンショット、PoC等の添付

## 対処スケジュール

- 受領後4日以内に受領確認を行います。
- アドバイザリ内で非公開にて協議し、修正・検証・公開タイミングを調整します。
- 修正でき次第、パッチとともにセキュリティアドバイザリを公開する予定です。

## 公開とクレジット

- 調整公開 (Coordinated Disclosure) を基本とします。
- 報告者にクレジットを付与する場合は、公開前に同意を得ます。

## プライバシーについて

- 報告者の氏名や連絡先は同意なく公開いたしません。ただし、法令等により開示が必要な場合は開示いたします。

## 謝辞

責任ある報告に感謝いたします。
